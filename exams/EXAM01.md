# بخش اول – سؤالات تشریحی

### سؤال تشریحی 1  
ماهیت نرم‌افزار را با تمرکز بر ویژگی‌های ناملموس بودن، عدم فرسودگی فیزیکی، و چرخهٔ تکاملی آن توضیح دهید. سپس تفاوت این ویژگی‌ها را با سخت‌افزار تحلیل کنید و نمونه‌ای واقعی برای هر تفاوت ارائه نمایید.

‍‍‍``` 1- نرم افزار بر خلاف نرم افزار فیزیکی نیست و در واقع ساختاری از داده ها و دستورات است همین ویژهگی باعث میشود  که تحلیل کیفیت آن دشوار تر باشد``` 

`2-نرم افزار مستهک نمیشود ولی با تغییرات پی در پی ساختار ان بی نظم و مستعد خطا میگردد ```
```3-نرم افزار از لحظه تحویل تا پایان عمر خود در چرخه ای از بازنگری اصلاح و افزودن ویژه گی های جدید قرار دارد``` 
```4-در نرم افزار مفهومی مشابه spare partوجود ندارد هر نقطه ریشه در طراحی یا پیاده سازی دارد و نیاز به اصلاح منطقی دارد``` 
5-توسعه نرم افزار بیشتر فرایند اتدیشه طراحی و تحقق منطق است نه مونتاژ مواد فیزیکی```

 در سخت افزار:فرسودگی فیزیکی طی زمان رخ میدهد مثل کاهش عمر باتری``` 
 ```چرخه تکاملی: نرم افازر پویا است و تحت یک چرخه تکاملی مستمر قرار دارد این چرخه شامل توسعه استمرار نگهداری و ارتقا است```
```سخت افزار:چرخه عمر محدودتر ``` 


### سؤال تشریحی 2  
هفت اصل عمومی مهندسی نرم‌افزار در فصل ۱ مطرح شده‌اند (مدلسازی قبل از ساخت، دید سیستمی، حفظ کیفیت، مواجهه با تغییر، طراحی برای تغییر، برنامه‌ریزی برای استفاده مجدد، و تفکر پیش از عمل). هر اصل را در یک جمله تعریف کرده و سپس دو اصل را انتخاب کنید و کاربرد آن‌ها را در یک پروژهٔ نرم‌افزاری واقعی شرح دهید.

1-مدل ساز ی ذهن را وادار میکند تا مسِِیله را قبل از پیاده سازی بفهمد``` 
2-هر سیستم جزِیی از سیستم بزرگ تر است طراحی باید مرزها وتعاملات را ببیند```
3-کیفیت با تست نهایی حاصل نمیشود از روز اول باید مهندسی کیفیت رعایت گردد ```
4-فرایند توسعه باید قابل تطبیق و انعطاف پذیر طراحی شود``` 
5-از ابتدا ساختار به گونه ایی انتخاب شود که افزودن ویژه گی  جدید یا حذف بخشها آسان باشد``` 
6-اجزای قابل استفاده مجدد زمان و هزینه را در اینده کاهش میدهد``` 

7-مهم ترین اصل پیش از عمل فکر کن برنامه ریزی و تحلیل پیشینی خطا را کاهش می دهد و تجربه می افریند```  

```مدل سازی قبل از ساخت : بیش از نوشتن حتی یک خط کد برای سیستم تیم توسعه دهنده از نمودارهای دیاگرام مانند نمودار موارد استفاده برای شناسایی کامل نقش ها دانشجو و استاد و مدیر و قابلیت اصلی برگزاری ازمون جریان تعامل با کاربر 
```تغییر طراحی برای تغییر : با توجه به ماهیت پویا و درحال تکامل سیستم را به گونه ای محسط اموزشی و  طراحان بین مدولار و ارتباط  سست بخش ها طراحی می کنند برای نمونه ماژول ارزیابی و ازمون به گونه ای مستقل از ماژول محتوا طراحی می شود ```

# بخش دوم – سؤالات تستی (چهارگزینه‌ای)

### سؤال 1  
کدام گزینه بهترین توصیف از ماهیت نرم‌افزار است؟  
- [ ] الف) مجموعه‌ای از قطعات فیزیکی که قابل جایگزینی هستند  
- [x] ب) ساختاری از داده‌ها و دستورات بدون ماهیت فیزیکی  
- [ ] ج) محصولی که به‌صورت طبیعی فرسوده می‌شود  
- [ ] د) مجموعه‌ای از مدارها و اتصالات  

### سؤال 2  
پدیدهٔ «فرسودگی نرم‌افزار» ناشی از چیست؟  
- [ ] الف) استهلاک سخت‌افزاری  
- [ ] ب) خرابی‌های الکتریکی  
- [x] ج) پیچیدگی افزوده‌شونده و تغییرات نامنظم  
- [ ] د) کاهش ظرفیت CPU  

### سؤال 3

- [ ] الف) سیستم‌های اطلاعاتی  
- [ ] ب) سیستم‌های مهندسی و علمی  
- [x] ج) سیستم‌های بلادرنگ  
- [ ] د) محصولات تولیدی  

### سؤال 4

- [ ] الف) سیستم حسابداری  
- [x] ب) نرم‌افزار شبیه‌سازی سازه  
- [ ] ج) اپلیکیشن فروشگاهی  
- [ ] د) سامانهٔ مدیریت مشتری  

### سؤال 5  
اصل «مدلسازی قبل از ساخت» بر چه نکته‌ای تأکید دارد؟  
- [ ] الف) شروع کدنویسی بدون تحلیل  
- [ ] ب) اولویت تست نهایی بر طراحی  
- [x] ج) فهم دقیق مسئله قبل از پیاده‌سازی  
- [ ] د) حذف نیازمندی‌ها از ابتدای پروژه  

### سؤال 6  
اصل «دید سیستمی» بیان می‌کند که:  
- [ ] الف) هر سیستم مستقل از محیط بیرونی است  
- [x] ب) هر سیستم بخشی از یک کل بزرگ‌تر است  
- [ ] ج) کیفیت فقط در مرحلهٔ تست ایجاد می‌شود  
- [ ] د) تغییرات اجتناب‌پذیر هستند

- [ ] ب) Reliability  
- [x] ج) Maintainability  
- [ ] د) Consistency  

### سؤال 8  
کدام عبارت دربارهٔ «طراحی برای تغییر» صحیح است؟  
- [ ] الف) افزودن ویژگی جدید باید از ابتدا دشوار باشد  
- [x] ب) ساختار سیستم باید از ابتدا قابل توسعه باشد  
- [ ] ج) طراحی تنها برای نسخهٔ اولیه انجام می‌شود  
- [ ] د) نیازمندی‌ها در طول پروژه ثابت‌اند
